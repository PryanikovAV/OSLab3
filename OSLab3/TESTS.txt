1. Тесты для Round Robin (RR)

1.1 Справедливое распределение процессорного времени
Описание:
- несколько процессов с одинаковым burstTime, но с разными arrivalTime.
- каждый процесс получает одинаковое время выполнения, независимо от arrivalTime.

Ожидаемый результат:
- каждый процесс должен чередоваться через равные интервалы времени.


1.2 Проблема частого переключения процессов (Context Switching)
Описание:
- большой burstTime, малый квант времени.

Ожидаемый результат:
- увеличится частота переключений, а следовательно, и системные накладные расходы.


1.3 Задержка для коротких процессов
Описание:
- несколько процессов с длинным burstTime,  и один с коротким burstTime.

Ожидаемый результат:
- короткий процесс может не завершиться сразу, так как будет чередоваться с другими процессами. Этот тест показывает недостаток RR в том, что короткие процессы могут задерживаться.

2. Тесты для SJF с вытеснением (SJF Preemptive)

2.1 Оптимизация для коротких процессов

Описание:
- запустите несколько процессов, чередующихся по burstTime, с разным arrivalTime (например, короткий процесс приходит чуть позже длинного).

Ожидаемый результат:
- алгоритм должен вытеснять текущие процессы ради завершения коротких. Это подчеркивает эффективность SJF для минимизации времени выполнения коротких задач.

2.2 Сценарий "Process Starvation"

Описание:
- добавьте процессы с длинным burstTime, пока продолжают приходить короткие процессы.

Ожидаемый результат:
- долгие процессы могут ожидать слишком долго, если короткие процессы продолжают приходить, что приводит к возможному "голоданию" длинных процессов.

2.3 Изменение выполняющегося процесса

Описание:
- добавьте процессы, которые приходят с меньшим burstTime во время выполнения другого процесса.

Ожидаемый результат:
- SJF должен быстро переключаться на новые, короткие процессы, демонстрируя приоритет коротких задач перед длительными.
